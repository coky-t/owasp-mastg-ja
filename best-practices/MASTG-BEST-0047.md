---
title: 継続的なアンチデバッグチェック (Continuous Anti-Debugging Checks)
alias: continuous-anti-debugging-checks
id: MASTG-BEST-0047
platform: android
knowledge: [MASTG-KNOW-0028]
---

機密性の高い実行パスでは、起動時の一度限りのチェックに頼るのではなく、頻繁にアンチデバッグチェックを実装します。

一度限りのチェックは、攻撃者が初期化後にデバッガをアタッチして、初期検出ロジックをトリガーすることなく解析を継続できるため、バイパスが容易です。実行時にデバッガの状態を再チェックすることで、攻撃者の労力を増し、耐性を向上します。

Android では、脅威モデルに応じて Java とネイティブのチェックを組み合わせることができます。たとえば、Java フローでは [`Debug.isDebuggerConnected()`](https://developer.android.com/reference/android/os/Debug#isDebuggerConnected()) を使用し、機密性の高い JNI パスではネイティブチェックで補完します。

さらに、頻繁なポーリングループはリソースを大量消費する可能性があります。適切なバランスは、機密性の高いアクション (たとえば、キーの使用、支払いの承認、特権 API 呼び出し、シークレットへのアクセスなど) の前にチェックポイントベースのチェックを実装し、機密性の高いフローがアクティブな間のみ短い周期的なチェックを実装することです。このモデルはオーバーヘッドを削減しつつ、起動後のデバッガアタッチメントを困難にします。

アンチデバッグは単独の粒度ではなく多層コントロールとして捉えます。高度な攻撃者は、パッチ適用、計装、フックを通じて依然としてチェックをバイパスする可能性があります。
